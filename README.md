# WMS Dispatch — 智能仓储调度管理系统

一个面向仓储运营场景的全栈调度管理平台，集成 AI Agent 实现自然语言驱动的工单调度与跨仓调拨，支持多角色协作。

## 功能概览

| 模块 | 核心能力 |
|------|----------|
| 账号与人员管理 | 多角色账号、工人技能标签、仓库归属 |
| 仓储与商品管理 | 仓库信息（含地理坐标）、商品档案、库存盘点 |
| 订单管理 | 订单创建、状态跟踪、明细管理 |
| 工单管理 | 拣货 / 备货 / 发货三类工单，手动与 AI 双路创建 |
| AI 工单调度 | 自然语言输入，Agent 自动匹配工人与技能，生成工单方案 |
| AI 跨仓调拨 | 自然语言描述库存缺口，Agent 给出调拨建议，走审批流 |
| 效率报告 | 管理员自定义时间范围，LLM 生成图表 + 文字分析报告 |
| 告警通知 | 超时工单与低库存自动触发站内通知 |

---

## 角色与权限

系统内置三种角色，权限严格隔离：

- **管理员（Admin）**：管理所有账号、仓库、商品、库存、订单；审批跨仓调拨申请；查看效率报告。
- **调度员（Dispatcher）**：查看并处理本仓库订单；创建、管理工单；使用 AI Agent 调度工单与发起调拨；接收告警通知。
- **工人（Worker）**：查看分配给自己的工单；更新工单执行状态；提交完工备注。

## 功能详情

### 管理员模块

#### 账号管理
- 创建、编辑管理员、调度员和工人账号。
- 为工人设置技能标签（支持多选，来自统一技能字典）和所属仓库。
- 禁用 / 启用账号，禁用后该账号无法登录。管理员账号不可被禁用。
- 账号支持设置头像和个人描述（可选）。

#### 仓库管理
- 创建、编辑、删除仓库信息，包括名称、地址、容量。
- 地址通过地图 API 解析并存储经纬度，供调度系统计算距离与路线使用。
- 仓库支持封面图和描述（可选）。

#### 商品管理
- 维护商品基础档案：SKU、名称、分类、单位重量。
- 为商品设置所需操作技能（多选），用于工单分配时的技能匹配。
- 商品支持封面图和描述（可选）。

#### 库存管理
- 按仓库查看各商品的可用库存、预留库存和库存阈值。
- 支持人工盘点修正库存数量。
- 当某仓库某商品库存低于阈值时，自动触发告警通知推送给对应调度员和管理员。

#### 订单管理
- 创建订单，指定客户、所属仓库、优先级、描述，并录入商品明细（商品 + 数量 + 单价）。
- 查看订单列表，跟踪订单状态（待处理 / 进行中 / 已完成 / 已取消）。
- 查看订单关联的所有工单进度。

#### 调拨审批
- 查看调度员或 AI Agent 发起的跨仓调拨申请列表。
- 审批通过或驳回，驳回时可填写原因。
- 审批结果自动通知发起人。

#### 效率报告
- 手动触发生成效率分析报告，支持自定义时间范围，或选择查看某人员的全部历史数据。
- 报告包含统计图表（工单完成率、平均完成时长、超时率、各类型工单分布等）和 LLM 生成的自然语言分析（异常识别、趋势解读、改进建议）。
- 报告生成后持久化存储，可随时回看历史报告。

### 调度员模块

#### 订单看板
- 查看本仓库所有待处理和进行中的订单列表，包含优先级、客户、商品明细、当前状态。

#### 工单看板
- 查看当前仓库所有工单的进度和状态，支持按任务类型（拣货 / 备货 / 发货）和状态筛选。

#### AI 工单调度
- 通过自然语言描述调度需求（例如："帮我把订单 #2024001 的拣货任务分配给空闲的工人"）。
- Agent 自动查询当前订单状态和空闲工人，结合技能匹配规则生成工单分配方案，并说明分配理由。
- 调度员审阅方案后确认或驳回；确认后工单正式创建并通知工人。
- 调度员也可手动创建工单，填写任务类型、描述、优先级，手动指定工人。

#### AI 跨仓调拨
- 通过自然语言描述库存缺口（例如："仓库A 的商品X 快不够了，帮我看看哪个仓库可以调"）。
- Agent 自动查询各仓库库存余量，综合分析后给出调拨建议（来源仓库、调拨数量、理由）。
- 调度员确认后生成调拨申请单，进入管理员审批流。

#### 告警通知
- 接收系统推送的告警：工单超时未完成、负责商品库存低于阈值。
- 每条告警包含告警描述和处理建议，支持标记已读。

---

### 工人模块

#### 我的工单
- 查看所有分配给自己的工单列表，包含任务类型（拣货 / 备货 / 发货）、关联订单、商品信息、优先级和截止要求。

#### 工单操作
- 开始执行：将工单状态更新为进行中，记录开始时间。
- 完成确认：将工单标记为已完成，记录完成时间，并填写完工备注。
- 完工备注为结构化字段：选择备注类型（正常完成 / 货物破损 / 数量不符 / 其他）并填写补充说明。一张工单可提交多条备注。

---

### AI Agent 模块

#### 工单调度 Agent
- 接收调度员自然语言输入，通过**只读 SQL Tool** 查询当前订单状态和工人空闲情况。
- 调用技能匹配规则，筛选符合商品所需技能的可用工人。
- 调用 LLM 综合生成工单分配方案，附带分配理由，供调度员确认。

> ⚠️ **安全约束**：Agent 只拥有数据库的**只读访问权限**，所有写操作（创建工单、更新状态等）必须通过应用 API 层执行，确保经过完整的权限检查和业务规则校验。

#### 跨仓调拨 Agent
- 接收调度员自然语言输入，通过**只读 SQL Tool** 查询所有仓库相关商品的库存余量。
- 调用 LLM 综合分析，给出最优调拨来源和数量建议，生成调拨申请单。

## 快速开始

### 环境要求

- Python >= 3.11
- Node.js >= 18
- PostgreSQL >= 15
- 有效的 Qwen API Key

### 安装

```bash
git clone https://github.com/your-org/wms-dispatch.git
cd wms-dispatch
cp .env.example .env
# 填写 .env 中的数据库连接和 API Key

# 启动数据库（需要 Docker）
docker compose up -d

# 后端
cd backend
pip install -r requirements.txt
alembic upgrade head      # 执行数据库迁移
uvicorn main:app --reload # 启动开发服务器

# 前端
cd frontend
npm install
npm run dev
```

### 环境变量

参见 [.env.example](./.env.example)：

```env
DATABASE_URL=postgresql://user:password@localhost:5432/wms
QWEN_API_KEY=your_qwen_api_key
SECRET_KEY=your_secret_key
```

> ⚠️ **安全提示**：`SECRET_KEY` 应使用足够长度的随机字符串（建议 ≥ 32 位），不要使用默认值或弱密钥。

## 安全设计

| 安全领域 | 措施 |
|----------|------|
| 身份认证 | JWT Token（python-jose / bcrypt），Token 有效期控制 |
| 权限控制 | 基于角色的访问控制（RBAC），API 层按角色拦截请求 |
| 密码安全 | bcrypt 哈希存储，禁止明文传输和存储 |
| AI Agent 隔离 | Agent SQL Tool 仅限只读事务，写操作必须走应用 API |
| 输入校验 | Pydantic Schema 校验所有 API 入参，防止注入和非法数据 |
| 审计日志 | 关键操作写入 audit_logs 表，不可修改、不可删除 |
| 通信安全 | 生产环境强制 HTTPS，WebSocket 使用 WSS |

> 详细数据库安全设计见 [database.md](./docs/database.md) 中的「安全设计备注」。
