- **管理员用户管理模块（完整落库版）**
  - 后端采用标准 3 层架构重构（Router -> Service -> Schema/Model）
  - 实现用户列表分页、模糊搜索过滤
  - 支持新增与编辑用户，强制校验邮箱格式与角色仓库绑定规则
  - 实现基于角色的状态修改与管理员不可禁用的防呆拦截
- **客户与产品管理模块（完整落库版）**
  - 客户/产品后端 CRUD、分页搜索、批量软删除、状态启停接口
  - 产品图片上传、替换、移除全链路完成（数据库 URL + 资源文件同步）

  - 前端客户与产品页面落地，支持状态开关（软删除可见）
- **增强仓库管理模块（与库存管理联动）**
  - 仓库列表支持点击仓库名称跳转独立库存详情页
  - 库存详情页支持搜索（产品名称/SKU/类别）、展示产品图片与可用状态
  - 支持人工盘点修正（现存量 + 阈值 + 原因），并落库 stocktakes / inventory_movements
  - 支持进货入库（可搜索选择产品），并落库 stocktakes / inventory_movements
  - 增加禁用防护：禁用仓库不可进入库存作业，下架商品不可盘点修正/进货
- **订单中枢（仅管理员部分，需要新用到订单表与订单明细表）**
  - 订单列表展示(客户、订单号、仓库(可空)、优先级、状态、责任调度员(可空)等)，搜索（订单号/客户/状态），可以设置日期范围过滤
  - 订单详情页展示
  - 订单创建接口
  - 订单列表导出（csv,markdown,pdf）与订单详情导出（pdf）
- **调度员接单中心 / 我的订单 / 工作台实时化**
  - 新增调度员订单域接口：待接单列表与详情、接单动作、我的订单列表与详情、工作台汇总
  - 接单中心与我的订单页面均接入真实 API，按优先级三列展示，支持列内排序与实时搜索
  - 新增独立详情路由（接单中心详情 / 我的订单详情），并区分展示接单入口与阶段/工单概览
  - 调度员布局接入实时统计与系统状态（`/api/health` 轮询），侧边栏“我的订单”显示动态数量
  - 工作台订单队列调整为仅展示进行中订单；中部订单明细价格统一在表格中展示（元格式）
  - 工作台保留原始占位设计并显式标注“占位”，未实现区块使用图片占位组件
- **调度员接单一致性与库存中心增强**
  - 接单关键路径按库存并发规则补强：订单行锁 + 库存行锁、库存预留落库、库存流水写入（`inventory_movements`）
  - 接单失败支持返回结构化库存缺口（按 `available` 对比，返回商品级 shortage 明细）
  - 前端接单页新增库存校验过程提示；库存不足时弹窗展示缺口明细，并支持跳转库存中心
  - 新增调度员只读库存中心：侧边栏入口、独立路由与页面，保留仓库顶部宽图与描述展示，支持分页搜索
- **订单阶段与工单联动**
  - 严格按照[service_rule.md](./service_rule.md)中定义的阶段完成规则，将订单阶段与工单联动，实现订单阶段状态流转（自动完成 + 手动完成双路径）。
  - 实现了调度员端创建、终止工单的功能
  - 实现工人端接单功能
- **调度员订单取消与我的订单筛选改造（闭环）**
  - 后端新增 `POST /api/dispatcher/my-orders/{order_id}/cancel`，仅允许责任调度员取消 `in_progress` 订单。
  - 取消前强制校验不存在 `pending/in_progress` 工单；取消时事务内完成预留库存释放与库存流水写入（`reserve_release`）。
  - 取消成功写入订单取消字段（`cancelled_at/cancelled_by/cancellation_reason`），并向管理员写入通知记录。
  - 前端“我的订单详情”新增取消按钮与取消原因弹窗；存在未终止工单时前置拦截。
  - “我的订单”筛选条支持 `进行中/已完成/已取消`，移除“隐藏已完成”开关。
  - 侧边栏移除“已完成订单”独立入口，统一为“我的订单”单页筛选模型。
- **调度员派单风控与工人端体验优化（进行中）**
  - 新增派单预校验接口：返回阶段技能最小/最大要求、工人当前阶段技能、在途工单负载、风险清单与商品级技能匹配明细。
  - 派单技能规则升级：
    - 低于当前阶段最低技能要求：禁止派单（前端工人下拉过滤 + 后端硬拦截）。
    - 介于最低与最高要求之间：标记技能风险，可强制派单。
    - 高于或等于最高要求：技能维度无风险。
  - 派单负载规则：在途工单数达到阈值（配置项）触发风险，允许强制派单。
  - 强制派单审计：风险放行时将结构化前缀写入工单备注，支持工人端详情识别与展示。
  - 调度员派单体验优化：有风险时隐藏普通备注输入，改为风险提示 + 弹窗统一填写放行原因；工人下拉无可选项时给出明确提示。
  - 工人端工单页改为“可点击卡片 + 详情面板”模式，卡片显示核心信息，详情展示风险放行结构化信息与备注正文。
