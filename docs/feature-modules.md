# 功能模块说明

本文档承接根目录 README 的详细功能说明，聚焦业务模块与能力边界，避免 README 过长。

---

## 管理员模块

### 账号管理

- 创建、编辑管理员、调度员和工人账号。
- 调度员和工人需绑定所属仓库，用于确定"本仓库"数据范围。
- 为工人设置技能标签（支持多选，来自统一技能字典）。
- 禁用/启用账号，禁用后该账号无法登录。管理员账号不可被禁用。
- 支持批量禁用用户，管理员账号不可被批量禁用。
- 管理员账号详情可查看；编辑权限限制为“管理员仅可编辑自己的管理员账号”。
- 账号支持设置头像和个人描述（由对应的人员自己操作，非管理员操作）。

### 客户管理

- 创建、编辑客户信息，包括客户名称、联系方式、送货地址。
- 创建订单时从客户列表中选择关联客户。

### 仓库管理

- 创建、编辑、删除仓库信息，包括名称、地址、容量。
- 地址通过地图 API 解析并存储经纬度，供调度系统计算距离与路线使用。
- 仓库支持封面图和描述（可选）。

### 商品管理

- 维护商品基础档案：SKU、名称、分类、单位重量。
- 为商品设置所需操作技能（多选），用于工单分配时的技能匹配。
- 商品支持封面图和描述（可选）。

### 库存管理

- 按仓库查看各商品的可用库存、预留库存、调拨软锁定数量和库存阈值。
- 仓库管理列表支持点击仓库名称进入独立“仓库库存详情页”，可按商品名称 / SKU / 类别搜索库存项。
- 库存详情页展示商品可用状态、封面图、现存量、预留量、锁定量、可用量与阈值。
- 支持人工盘点修正库存（现存量与阈值），可填写修正原因；修正会记录到盘点与库存流水。
- `qty_available` 由数据库生成列自动计算（`GENERATED ALWAYS AS (qty_on_hand - qty_reserved - qty_locked) STORED`），盘点修改现存量后会自动刷新，业务侧不直接写该字段。
- 支持“进货”操作：管理员可搜索并选择现有商品进行入库，系统自动记录流水。
- 禁用仓库不允许进入库存作业；下架商品不允许盘点修正或进货。
- 当某仓库某商品库存低于阈值时，自动触发告警通知推送给对应调度员和管理员。

### 订单管理

- 创建订单，指定客户、所属仓库、优先级、描述，并录入商品明细（商品 + 数量 + 单价）。
- 查看订单列表，跟踪订单状态（待接单/进行中/已完成/已取消）。
- 查看订单关联的所有工单进度。
- 取消订单规则：待接单状态可直接取消；进行中状态须先由管理员逐一终止所有工单后方可取消，取消时系统自动释放预留库存并通知责任调度员。

### 调拨审批

- 查看调度员或 AI Agent 发起的跨仓调拨申请列表。
- 审批通过前系统自动校验来源仓库当前库存是否仍满足调拨数量；若库存已变化，提示管理员选择部分调拨或驳回。
- 审批通过后立即扣减来源仓库库存，并在目标仓库生成待入库记录，等待调度员手动确认入库。
- 驳回时可填写原因；审批结果自动通知发起人。

### 效率报告

- 手动触发生成效率分析报告，支持自定义时间范围，或选择查看某人员的全部历史数据。
- 报告包含统计图表（工单完成率、平均完成时长、超时率、各类型工单分布等）和 LLM 生成的自然语言分析（异常识别、趋势解读、改进建议）。
- 报告生成后持久化存储，可随时回看历史报告。

## 调度员模块

### 订单看板与接单

- 查看本仓库所有待接单和进行中的订单列表，包含优先级、客户、商品明细、当前状态及责任调度员。
- 调度员可对待接单订单执行接单操作，接单即认领为责任人。接单时系统校验库存是否充足，库存不足则拦截并提示，调度员需先发起跨仓调拨补充库存后再接单。
- 接单成功后系统按订单商品明细全额预留库存，订单进入进行中状态。
- 进行中的订单对本仓库所有调度员可见，但仅责任调度员可操作；其他调度员只读。
- 接单后 24 小时内未创建任何工单，系统自动释放预留库存、将订单回退为待接单，并推送告警给责任调度员和管理员。
- 查看本仓库待入库记录列表；实物到达后手动确认入库，确认后目标仓库库存正式增加。待入库记录超时未确认时触发提醒。

### 工单看板

- 查看当前仓库所有工单的进度和状态，支持按任务类型（拣货/备货/发货）和状态筛选。

### AI 工单调度

- 通过自然语言描述调度需求（如：帮我把订单的拣货任务分配给空闲工人）。
- Agent 仅查询当前调度员已接单的订单，结合技能匹配规则和工人空闲情况生成工单分配方案，并说明分配理由。
- 调度员审阅方案后确认或驳回；确认后工单正式创建并通知工人。
- 调度员也可手动创建工单，填写任务类型、描述、优先级、截止时间，手动指定工人。
- 同一阶段可创建多张工单，分配给不同工人并行执行。

### AI 跨仓调拨

- 通过自然语言描述库存缺口（如：仓库 A 的商品 X 快不够了）。
- Agent 查询各仓库可调拨库存（可用库存减去已预留和软锁定数量），综合分析后给出调拨建议（来源仓库、调拨数量、理由）。
- 调度员确认后生成调拨申请单，系统对来源仓库该商品创建软锁定，进入管理员审批流。

### 告警通知

- 接收系统推送的告警：工单超时未完成、负责商品库存低于阈值、工单出现货物破损或数量不符异常备注。
- 每条告警包含告警描述和处理建议，支持标记已读。

## 工人模块

### 我的工单

- 查看所有分配给自己的工单列表，包含任务类型（拣货/备货/发货）、关联订单、商品信息、优先级和截止时间。

### 工单操作

- 开始执行：将工单状态更新为进行中，记录开始时间。
- 完成确认：将工单标记为已完成，记录完成时间，并填写完工备注。
- 完工备注为结构化字段：选择备注类型（正常完成/货物破损/数量不符/其他）并填写补充说明。一张工单可提交多条备注。
- 提交货物破损或数量不符类型备注时，系统自动触发告警推送给责任调度员和管理员。其他类型备注只通知责任调度员。

## AI Agent 模块

### 工单调度 Agent

- 接收调度员自然语言输入，通过只读 SQL Tool 查询当前调度员已接单的订单状态和工人空闲情况。
- 调用技能匹配规则，筛选符合商品所需技能的可用工人。
- 调用 LLM 综合生成工单分配方案，附带分配理由，供调度员确认。

安全约束：Agent 只拥有数据库的只读访问权限，所有写操作（创建工单、更新状态等）必须通过应用 API 层执行，确保经过完整权限检查和业务规则校验。

### 跨仓调拨 Agent

- 接收调度员自然语言输入，通过只读 SQL Tool 查询所有仓库相关商品的可调拨库存（可用库存减去已预留和软锁定数量）。
- 调用 LLM 综合分析，给出最优调拨来源和数量建议，生成调拨申请单。
